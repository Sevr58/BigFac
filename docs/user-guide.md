## 1. Что это такое

**Social Content Factory (SCF)** — платформа для автоматизации создания и публикации контента в социальных сетях. Один человек может управлять контентом для нескольких брендов одновременно.

**Что умеет система:**

| Функция | Описание |
|---------|----------|
| Стратегия | Claude генерирует контент-пиллары и 30-дневный план |
| Репурпозинг | Загружаешь видео/аудио — система нарезает клипы, извлекает цитаты, создаёт посты |
| Мультисеть | Instagram, VK, Telegram — каждый в своём формате |
| Воркфлоу | Черновик → согласование → публикация |
| Публикация | Автоматическая отложенная публикация по расписанию |
| Аналитика | Статистика по постам + AI-рекомендации от Claude |

**Стек:**
- Backend: FastAPI, PostgreSQL, Redis, Celery
- AI: Claude (Anthropic), OpenAI Whisper (транскрипция)
- Frontend: Next.js, Tailwind CSS
- Хранилище: локальная папка или Yandex Object Storage / AWS S3

## 2. Требования и установка

### Системные требования

- Docker Desktop (рекомендуется) **или** Python 3.12+ и Node.js 20+
- 4 ГБ RAM минимум (обработка видео требует больше)
- FFmpeg — установить на хост (не через Docker)

### API-ключи (обязательно)

| Ключ | Где получить | Для чего |
|------|-------------|---------|
| `ANTHROPIC_API_KEY` | https://console.anthropic.com | Генерация контента, стратегия, аналитика |
| `OPENAI_API_KEY` | https://platform.openai.com | Транскрипция видео через Whisper |
| `TAVILY_API_KEY` | https://tavily.com | Веб-поиск для контекста черновиков |

### Клонирование и настройка

```bash
# 1. Перейти в папку проекта
cd BigFac

# 2. Создать .env файл для бэкенда
cp backend/.env.example backend/.env

# 3. Заполнить .env (минимум обязательных полей):
```

**`backend/.env` — что заполнить:**


## 3. Запуск и первый старт

### Вариант A: Docker Compose (рекомендуется)

```bash
# Запускает PostgreSQL, Redis и API одной командой:
cd BigFac
docker-compose up

# Первый раз применить миграции (новый терминал):
docker-compose exec api alembic upgrade head
```

### Вариант B: Ручной запуск

**Шаг 1 — База данных и Redis**
```bash
# Убедись, что PostgreSQL и Redis запущены локально
# Создай базу данных:
createdb scf
```

**Шаг 2 — Бэкенд**
```bash
cd backend
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**Шаг 3 — Celery Worker** (новый терминал — обязательно для обработки файлов и публикаций)
```bash
cd backend
celery -A app.worker:celery_app worker --loglevel=info
```

**Шаг 4 — Celery Beat** (ещё один терминал — для автопубликации по расписанию)
```bash
cd backend
celery -A app.worker:celery_app beat --loglevel=info
```

**Шаг 5 — Фронтенд** (новый терминал)
```bash
cd frontend
npm install
npm run dev
```

| URL | Что открывает |

## 4. Регистрация и вход

### Регистрация

1. Открой http://localhost:3000/register
2. Введи email, пароль (минимум 8 символов), полное имя
3. Нажми **Зарегистрироваться**
4. Тебя автоматически перенаправит на онбординг

### Вход

1. Открой http://localhost:3000/login
2. Введи email и пароль
3. JWT-токен сохраняется в localStorage и автоматически прикрепляется ко всем запросам

> **Срок жизни токена:** 7 дней. После истечения нужно войти снова.

## 5. Онбординг: создание воркспейса и бренда

При первом входе система показывает форму онбординга. Это **одноразовый шаг** — потом данные можно изменить в разделе «Бренд».

### Что нужно заполнить

**Шаг 1 — Название компании / бренда**
- Введи название (например: «Кофейня Грин», «Агентство Максим»)

**Шаг 2 — Тип компании**
- SMB (малый бизнес)
- Enterprise
- Agency
- Creator (персональный бренд)

**Шаг 3 — Описание**
- Несколько предложений о чём компания, что продаёт, в чём уникальность
- *Чем подробнее, тем лучше контент генерирует Claude*

**Шаг 4 — Целевая аудитория**
- Опиши: кто покупает, возраст, интересы, боли
- Например: «Женщины 25-40, Москва, любят кофе, ценят качество»

**Шаг 5 — Цели**
- Выбери несколько: awareness, leads, sales, retention, community

**Шаг 6 — Социальные сети**
- Поставь галочки для нужных сетей: Instagram, VK, Telegram
- Можно включить все три или только одну

**Шаг 7 — Тон голоса**
- neutral / friendly / professional / casual / inspiring

**Шаг 8 — Частота публикаций**
- daily / 3x_week / weekly

После заполнения нажми **Сохранить** — создастся воркспейс и профиль бренда.

## 6. Генерация контент-стратегии

### Где найти

Навигация → **Стратегия**

### Как сгенерировать

1. Нажми кнопку **Сгенерировать стратегию**
2. Claude анализирует профиль бренда и создаёт:
   - **5+ контент-пилларов** (темы для постов с описанием и воронкой)
   - **30-дневный контент-план** (конкретные посты по сетям, форматам и стадиям воронки)
3. Результат появляется на странице через 15-30 секунд

### Что получишь

**Контент-пилларс** — основные рубрики контента. Каждый пиллар содержит:
- Название и описание
- К каким стадиям воронки относится (tofu/mofu/bofu/retention)

**Контент-план** — таблица постов с полями:
- Сеть (Instagram, VK, Telegram)
- Формат (reel, post, story, carousel)
- Стадия воронки (TOFU, MOFU, BOFU, Retention)
- Тема поста
- Запланированная дата

> **Совет:** Если стратегия не устраивает — нажми «Сгенерировать» ещё раз. Каждый раз Claude создаёт новый вариант.

### Стадии воронки

| Стадия | Назначение | Примеры форматов |
|--------|-----------|-----------------|
| **TOFU** — Awareness | Привлечение новой аудитории | Reels, educational posts |
| **MOFU** — Consideration | Прогрев, объяснение ценности | Карусели, сравнения, кейсы |
| **BOFU** — Conversion | Призыв купить | Офферы, отзывы, CTA |
| **Retention** — Лояльность | Удержание клиентов | Закулисье, UGC, обновления |

## 7. Загрузка и обработка ассетов

### Где найти

Навигация → **Ассеты**

### Поддерживаемые форматы

| Тип | Расширения | Что происходит |
|-----|-----------|---------------|
| **Видео** | .mp4, .mov, .avi | Транскрипция + нарезка клипов + извлечение атомов |
| **Аудио** | .mp3, .wav, .m4a | Транскрипция + извлечение атомов |
| **Изображение** | .jpg, .png | Сохранение |
| **Текст** | .txt, .pdf | Извлечение атомов |

### Процесс загрузки

**1. Загрузи файл**
- Нажми **Загрузить ассет**
- Выбери файл (максимальный размер зависит от настроек хранилища)
- Подтверди загрузку

**2. Автоматическая обработка** (занимает 2-10 минут)

Celery worker выполняет последовательно:
```
Загрузка → Транскрипция (Whisper) → Определение сцен (PySceneDetect)
→ Нарезка клипов (FFmpeg) → Анализ атомов (Claude)
```

Статусы ассета: `uploaded` → `processing` → `ready` (или `failed`)

**3. Просмотр результата**

После обработки ассет покажет:
- **Транскрипция** — полный текст из видео/аудио
- **Атомы контента** — выжимки, которые использует Claude при генерации черновиков:
  - `hook` — цепляющие зачины
  - `key_point` — ключевые тезисы
  - `quote` — цитаты
  - `cta` — призывы к действию
  - `story` — истории
  - `clip` — ссылки на видеоклипы

> **Зачем нужны ассеты?** При генерации черновика на основе ассета Claude использует реальный контент из видео — транскрипцию и атомы — вместо выдумки. Это даёт точный, брендированный текст.

## 8. Создание черновиков

### Где найти

Навигация → **Черновики**

### Два способа создать черновик

#### Способ 1 — Репурпозинг из ассета (рекомендуется)

1. Нажми **Создать черновик**
2. Выбери **Сеть** (Instagram / VK / Telegram)
3. Выбери **Формат** (reel / post / story / carousel / thread)
4. Выбери **Стадию воронки** (tofu / mofu / bofu / retention)
5. Выбери **Ассет** из библиотеки (видео или аудио со статусом `ready`)
6. Нажми **Сгенерировать**

Claude получает транскрипцию и атомы ассета → создаёт текст поста с хэштегами.

#### Способ 2 — Исследовательский путь (без ассета)

1. Выполни шаги 1-4 как выше
2. НЕ выбирай ассет
3. Нажми **Сгенерировать**

Tavily ищет контекст в интернете → Claude использует его для генерации.

### Статусы черновиков

```
draft → needs_review → approved → scheduled → publishing → published
                    ↘ rejected → (правка) → needs_review
                                                         ↘ failed
```

### Редактирование черновика

1. Найди черновик в списке
2. Нажми на него → откроется редактор
3. Отредактируй **текст** и **хэштеги**
4. Сохрани — создаётся новая версия (история версий сохраняется)

### Отправка на согласование

После того как черновик готов:
1. Нажми **Отправить на ревью**
2. Статус изменится на `needs_review`
3. Черновик появится в очереди согласования

## 9. Approval Workflow — Согласование публикаций

Система требует явного одобрения перед публикацией. Это защита от случайного выхода некачественного контента.

### Роли

| Роль | Может создавать | Может согласовывать | Может публиковать |
|------|----------------|--------------------|--------------------|
| **owner** | ✅ | ✅ | ✅ |
| **editor** | ✅ | ❌ | ❌ |
| **approver** | ❌ | ✅ | ❌ |

### Процесс согласования (approver / owner)

1. Перейди в раздел **Черновики** — вкладка «На согласование»
   или через API: `GET /api/v1/approvals/queue?brand_id=ID`
2. Открой черновик, прочитай текст
3. Действия:
   - **Одобрить** → статус `approved`, можно планировать публикацию
   - **Отклонить** (с комментарием) → статус `rejected`, редактор получает фидбек

### Что происходит после отклонения

1. Редактор видит черновик в статусе `rejected` с комментарием
2. Редактирует текст
3. Снова отправляет на ревью (`needs_review`)
4. Цикл повторяется до одобрения

## 10. Human Tasks — Задачи для людей

Некоторые задачи нельзя автоматизировать: снять видео, сделать фото, записать голос. Для этого есть система задач.

### Где найти

Навигация → **Задачи**

### Как создать задачу

1. Нажми **Новая задача**
2. Заполни:
   - **Название** (например: «Записать видео про новое меню»)
   - **Описание** (детали, что именно снять, требования)
   - Опционально: привяжи к черновику (`draft_id`)
3. Сохрани

### Статусы задач

| Статус | Значение |
|--------|---------|
| `pending` | Создана, ждёт исполнения |
| `in_progress` | Кто-то взял в работу |
| `completed` | Выполнена (можно прикрепить результирующий ассет) |
| `cancelled` | Отменена |

### Завершение задачи

При завершении можно указать `result_asset_id` — ID загруженного ассета (например, снятого видео). Это связывает задачу с результатом.

### Типичный сценарий

1. Claude сгенерировал черновик Reel
2. Система (или менеджер) создала задачу: «Записать рилс по тексту черновика #42»
3. Оператор снял видео, загрузил в ассеты
4. Отметил задачу выполненной, прикрепил ассет
5. Видео обработалось → менеджер использовал ассет для финального черновика

## 11. Настройка социальных сетей

Чтобы система могла публиковать, нужно прописать credentials для каждой сети.

### Где настроить

API: `PATCH /api/v1/workspaces/{workspace_id}/brand/social-accounts/{account_id}/credentials`

(Фронтенд-интерфейс для этого — в разделе **Бренд**, социальные аккаунты)

### Telegram

**Что нужно:**
1. Создай бота через @BotFather в Telegram
2. Получи `bot_token` (вида `1234567890:AABBccddeeff...`)
3. Добавь бота в канал как администратора
4. Узнай `chat_id` канала (например, через @username_to_id_bot или API)

**Credentials JSON:**
```json
{
  "bot_token": "1234567890:AABBccddeeff...",
  "chat_id": "-1001234567890"
}
```

> **chat_id для каналов** всегда начинается с `-100`. Для личных чатов — положительное число.

### VK

**Что нужно:**
1. Зайди в https://vk.com/dev → создай Standalone-приложение
2. Получи токен с правами `wall`, `offline`
3. Узнай `owner_id` (для группы — отрицательное число `-GROUP_ID`)

**Credentials JSON:**
```json
{
  "access_token": "vk1.a.xxxxx...",
  "owner_id": "-123456789"
}
```

### Instagram (Meta Graph API)

**Что нужно:**
1. Бизнес-аккаунт Instagram + привязанная Facebook-страница
2. Создай приложение в https://developers.facebook.com
3. Получи `Page Access Token` с правами `instagram_basic`, `instagram_content_publish`
4. Узнай `Instagram Account ID` через Graph API Explorer

**Credentials JSON:**
```json
{
  "page_access_token": "EAAxxxx...",
  "instagram_account_id": "17841234567890"
}
```

> **Ограничение MVP:** Instagram публикует только посты с изображением (IMAGE container). Публикация без медиа через Instagram API не поддерживается платформой.

### Как прописать credentials через API

```bash
curl -X PATCH "http://localhost:8000/api/v1/workspaces/1/brand/social-accounts/1/credentials" \
  -H "Authorization: Bearer ВАШ_ТОКЕН" \
  -H "Content-Type: application/json" \
  -d '{"credentials": {"bot_token": "...", "chat_id": "..."}}'
```

## 12. Планирование и публикация

### Где найти

Навигация → **Публикации**

### Планирование поста

1. Перейди в раздел **Публикации**
2. В блоке «Одобренные и запланированные» найди нужный черновик (статус `approved`)
3. Выбери дату и время публикации через datetime-picker
4. Нажми **Запланировать**
5. Статус черновика изменится на `scheduled`, появится время публикации

> **Требование:** Планировать можно только черновики со статусом `approved`.

### Отмена публикации

1. Найди пост со статусом `scheduled`
2. Нажми **Отменить**
3. Статус вернётся в `approved`

### Как происходит автопубликация

Каждые **60 секунд** Celery Beat проверяет:
- Все черновики со статусом `scheduled`
- Для которых `scheduled_at <= текущее_время`

Если нашёл — ставит в очередь задачу `publish_post`:

```
scheduled → publishing → [вызов API сети] → published (успех)
                                           ↘ failed (ошибка)
```

**Retry-логика:** При ошибке публикации система повторит попытку 3 раза:
- 1-я попытка сразу
- 2-я через 60 секунд
- 3-я через 120 секунд
- 4-я через 240 секунд
- Если все неудачны — статус `archived`, ошибка логируется

### История публикаций

В нижней части страницы «Публикации» — таблица всех опубликованных постов:
- Сеть
- ID поста в сети (для перехода к посту)
- Дата публикации
- Статус (Успешно / Ошибка)

### UTM-метки

Каждый опубликованный пост автоматически получает UTM-параметры:

| Параметр | Значение |
|----------|---------|
| `utm_source` | Название сети (instagram, vk, telegram) |
| `utm_medium` | Формат поста (reel, post, story) |
| `utm_campaign` | Всегда `scf` (Social Content Factory) |
| `utm_content` | Стадия воронки (tofu, mofu, bofu, retention) |
| `utm_term` | Slug названия бренда |

UTM-метки хранятся в записи `PublishedPost` и используются для атрибуции лидов.

## 13. Аналитика и AI-фидбек

### Где найти

Навигация → **Аналитика**

### Сводка (Summary Cards)

Четыре карточки вверху страницы:
- **Постов опубликовано** — общее количество
- **Просмотры** — суммарно по всем постам
- **Лиды** — количество зафиксированных лид-событий
- **Активных сетей** — в скольких сетях есть публикации

### Графики

**По сетям (просмотры)** — горизонтальные бары, сравнивает эффективность Instagram vs VK vs Telegram.

**По форматам (просмотры)** — сравнивает reel vs post vs story vs carousel.

**По воронке (постов)** — показывает распределение контента по TOFU/MOFU/BOFU/Retention.

### Топ постов

Таблица с 5 лучшими постами по просмотрам. Показывает:
- Сеть
- Дата публикации
- Просмотры, лайки, репосты

### Как собираются метрики

**Автоматически** — каждый день в 3:00 по московскому времени Celery Beat запускает `collect_all_metrics`:
- **VK** — запрашивает `wall.getById` для каждого опубликованного поста
- **Instagram** — запрашивает Graph API insights
- **Telegram** — метрики недоступны через Bot API (нужен TGStat или Telegram Analytics)

Метрики собираются только для постов возрастом до **30 дней**.

### Регистрация лид-события

Когда пользователь переходит по ссылке с UTM и совершает целевое действие, можно зафиксировать лид через API:

```bash
curl -X POST "http://localhost:8000/api/v1/analytics/leads" \
  -H "Authorization: Bearer ВАШ_ТОКЕН" \
  -H "Content-Type: application/json" \
  -d '{
    "brand_id": 1,
    "published_post_id": 42,
    "event_type": "lead",
    "utm_source": "instagram",
    "utm_medium": "reel",
    "utm_campaign": "scf",
    "utm_content": "bofu"
  }'
```

### AI-анализ стратегии (Feedback Loop)

1. Нажми **Запустить анализ** в блоке «AI-анализ стратегии»
2. Claude получает данные за последние 30 дней:
   - Количество постов по сетям и форматам
   - Суммарные просмотры, лайки, репосты
   - Распределение по воронке
   - Количество лидов
3. Claude анализирует, что работает, и выдаёт **3-5 конкретных рекомендаций** на русском языке

**Пример рекомендации:**
> «Reels в Instagram показывают в 3 раза больше охвата, чем статичные посты. Рекомендую увеличить долю Reels до 60% от плана и перенести часть MOFU-контента из VK в Instagram для расширения аудитории...»

## 16. Архитектура и фоновые процессы

### Схема системы

```
Браузер (Next.js :3000)
    ↓ HTTP запросы
FastAPI (:8000)
    ↓ ORM
PostgreSQL (:5432) ← все данные

FastAPI → Celery (через Redis)
Redis (:6379) ← очередь задач

Celery Worker ← обрабатывает задачи:
    ├── asset_tasks.py  (Whisper, PySceneDetect, FFmpeg, Claude)
    ├── draft_tasks.py  (Tavily, Claude)
    ├── publish_tasks.py (Telegram/VK/Instagram API)
    └── analytics_tasks.py (VK/IG metrics API)

Celery Beat ← расписание:
    ├── schedule_pending_posts — каждые 60 сек
    └── collect_all_metrics   — ежедневно в 03:00 МСК
```

### Обработка ассетов (детально)

```
1. User загружает файл → FastAPI сохраняет в storage (local /tmp/scf_uploads или S3)
2. process_asset.delay(asset_id) → Celery worker берёт задачу
3. Whisper API: транскрипция (язык: ru)
4. PySceneDetect: находит смены сцен (threshold=27.0)
5. FFmpeg: нарезает до 10 клипов по сценам
6. Claude: анализирует транскрипцию → 5-10 атомов (hook/quote/cta/etc.)
7. Asset.status = "ready", transcription и atoms записаны
```

### Генерация черновиков (детально)

```
Если source_asset_id указан (Repurposing Path):
  1. Загружает атомы ассета из БД
  2. Формирует промпт: brand context + atoms + network/format/funnel params
  3. Claude → текст поста + хэштеги на русском

Если source_asset_id не указан (Research Path):
  1. Tavily: ищет релевантный контент в интернете по теме бренда
  2. Формирует промпт: brand context + search results + params
  3. Claude → текст поста + хэштеги на русском
```

## 17. Решение проблем

### Ошибка: celery worker не запущен

**Симптом:** Ассеты застряли в статусе `processing`, черновики не генерируются

**Решение:**
```bash
cd backend
celery -A app.worker:celery_app worker --loglevel=info
```

### Ошибка: пост не публикуется в срок

**Симптом:** Пост в статусе `scheduled`, время прошло, но он не опубликован

**Причина:** Celery Beat не запущен

**Решение:**
```bash
cd backend
celery -A app.worker:celery_app beat --loglevel=info
```

### Ошибка публикации в Telegram: "chat not found"

**Причина:** Неверный `chat_id` или бот не добавлен в канал

**Решение:**
1. Убедись, что бот добавлен в канал как администратор
2. Проверь `chat_id` — для каналов он отрицательный и начинается с `-100`
3. Обнови credentials через API

### Ошибка публикации в VK: "User authorization failed"

**Причина:** Истёк access_token

**Решение:**
1. Получи новый токен через VK OAuth
2. Обнови credentials через API

### Ошибка: "ImportError: No module named 'whisper'"

**Решение:**
```bash
cd backend
pip install openai-whisper
```

### Ошибка: FFmpeg не найден

**Симптом:** Нарезка клипов не работает, ассет падает в `failed`

**Решение (Windows):**
```powershell
winget install FFmpeg
# или скачай с https://ffmpeg.org/download.html и добавь в PATH
```

**Решение (Linux/Mac):**
```bash
sudo apt install ffmpeg  # Ubuntu
brew install ffmpeg      # macOS
```

### База данных не инициализирована

**Симптом:** `relation "users" does not exist`

**Решение:**
```bash
cd backend
alembic upgrade head
```

### Метрики не собираются для Telegram

**Это ожидаемое поведение.** Telegram Bot API не предоставляет аналитику по отдельным сообщениям. Для метрик Telegram нужны сторонние сервисы (TGStat, Telemetr).

### Проверка статуса задач Celery

```bash
# Посмотреть активные задачи:
celery -A app.worker:celery_app inspect active

# Посмотреть зарезервированные:
celery -A app.worker:celery_app inspect reserved
```

## Приложение A: Типичный рабочий день с SCF

### Утро (10 минут)

1. Открой http://localhost:3000
2. Зайди в **Стратегию** → посмотри контент-план на сегодня
3. Зайди в **Задачи** → что нужно снять/сделать сегодня?

### Создание контента (20-30 минут)

1. Открой **Черновики** → нажми **Создать**
2. Выбери сеть, формат, стадию воронки из плана
3. Если есть подходящий ассет — выбери его
4. Нажми **Сгенерировать** → подожди 15-30 сек
5. Прочитай текст, отредактируй при необходимости
6. Нажми **Отправить на ревью**

### Согласование (5 минут — approver)

1. Получи уведомление (или зайди в **Черновики**)
2. Прочитай пост
3. **Одобрить** или **Отклонить** с комментарием

### Планирование

1. Зайди в **Публикации**
2. Для каждого одобренного поста выбери время
3. Нажми **Запланировать**
4. Система автоматически опубликует в нужное время

### Конец недели (15 минут)

1. Зайди в **Аналитику**
2. Посмотри что сработало лучше всего
3. Нажми **Запустить анализ** → получи рекомендации Claude
4. Обнови стратегию под рекомендации
